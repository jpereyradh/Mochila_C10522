a. ¿Quién inventó el sistema de control de versión Git y por qué?
Git es un programa de control de versiones, un proyecto de código abierto y matenimiento 
activo diseñado por Linus Torvalds en 2005. El propósito de dicho proyecto fue pensado
para optimizar el rendimiento, la eficiencia, la confiabilidad y compatibilidad de versiones de aplicaciones que tienen un gran número de archivos de código fuente. El objetivo es llevar registro de los cambios en archivos de computadora incluyendo coordinar el trabajo que varias personas realizan sobre archivos compartidos en un repositorio de código.

b. ¿A quién pertenece actualmente Github y por qué?
GitHub permite alojar proyectos en la nube utilizando un sistema de control de versiones Git. Luego del mayor ataque de denegación de servicio (DDoS) de la historia que sufrió Github, ese mismo año Microsoft compró la compañía y hoy le pertenece.

c. ¿Hay otra forma que no sea la terminal para trabajar con Github?
Git Kraken es una herramienta multiplataforma que nos permite manejar Git y vincular el repositorio junto con su historial de cambios en Github a través de una interfaz gráfica bastante amigable para alguien que se está iniciando en el mundo de la programación
